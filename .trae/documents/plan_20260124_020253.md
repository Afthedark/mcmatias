## Plan de Acción: Mejorar Animaciones UI/UX del Sidebar para Transiciones de Módulos

### Contexto Actual
- El sidebar se renderiza dinámicamente desde [components.js](file:///d:/myProjects/mcmatias/frontend/js/components.js#L159-L196) usando `SIDEBAR_CONFIG`.
- Los enlaces de navegación se generan en `createNavItem()` sin animaciones específicas.
- El CSS actual en [styles.css](file:///d:/myProjects/mcmatias/frontend/css/styles.css#L233-L287) tiene transiciones básicas pero sin efectos de entrada/salida de módulos.

### Objetivo
Implementar animaciones fluidas y modernas cuando el usuario navega entre módulos del sidebar, compatibles con escritorio y móvil.

### 1. Animaciones de Navegación (Entrada/Salida de Módulos)
**Archivo:** `frontend/css/styles.css`

**Agregar clases CSS:**
- `.page-transition-out`: Animación de salida (fade out + slide up)
- `.page-transition-in`: Animación de entrada (fade in + slide down)
- `.sidebar-item-active`: Efecto visual cuando un item está activo

### 2. JavaScript para Control de Transiciones
**Archivo:** `frontend/js/components.js`

**Modificar `createNavItem()` para agregar manejo de clicks:**
```javascript
function createNavItem(href, icon, text, activePage) {
    const isActive = activePage === href ? 'active' : '';
    return `
        <li class="nav-item">
            <a href="${href}" class="nav-link ${isActive}" 
               onclick="handleModuleTransition(event, '${href}')" 
               title="${text}">
                <i class="bi ${icon}"></i>
                <span>${text}</span>
            </a>
        </li>
    `;
}
```

**Agregar función de manejo de transiciones:**
```javascript
/**
 * Handle smooth module transitions
 */
function handleModuleTransition(event, href) {
    event.preventDefault();
    
    const currentContent = document.querySelector('.content-wrapper');
    const sidebar = document.querySelector('.sidebar');
    
    // En móvil, cerrar sidebar después de click
    if (window.innerWidth <= 992) {
        sidebar.classList.remove('show');
        document.querySelector('.sidebar-overlay').classList.remove('show');
    }
    
    // Animación de salida
    currentContent.classList.add('page-transition-out');
    
    setTimeout(() => {
        // Navegar a la nueva página
        window.location.href = href;
    }, 300);
}

/**
 * Initialize page entry animation
 */
function initializePageAnimations() {
    const content = document.querySelector('.content-wrapper');
    if (content) {
        content.classList.add('page-transition-in');
        
        setTimeout(() => {
            content.classList.remove('page-transition-in');
        }, 300);
    }
}

// Llamar al cargar cada página
if (document.readyState === 'loading') {
    document.addEventListener('DOMContentLoaded', initializePageAnimations);
} else {
    initializePageAnimations();
}
```

### 4. Mejoras de Performance y UX
**Optimizaciones:**
- Usar `will-change` para animaciones suaves
- Implementar `prefers-reduced-motion` para accesibilidad
- Agregar indicadores de carga durante transiciones

```css
/* Respetar preferencias de reducción de movimiento */
@media (prefers-reduced-motion: reduce) {
    .page-transition-out,
    .page-transition-in,
    .nav-link::before,
    .nav-item {
        animation: none !important;
        transition: none !important;
    }
}

/* Optimización de rendimiento */
.page-transition-out,
.page-transition-in,
.nav-link::before {
    will-change: transform, opacity;
}
```
