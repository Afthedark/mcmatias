## Plan de Acción: Quitar Animación Cascada del Sidebar

### Contexto
Actualmente, el sidebar tiene una animación cascada implementada que hace que los items aparezcan secuencialmente con un delay escalonado. El usuario solicita eliminar esta animación específica.

### Objetivo
Eliminar la animación cascada (`slideInLeft`) de los items del sidebar mientras se mantienen las demás animaciones funcionando.

### Cambios a Realizar

**Archivo:** `frontend/css/styles.css`

**Eliminar las siguientes reglas CSS:**
```css
/* Animación de items del sidebar */
.nav-item {
    animation: slideInLeft 0.3s ease-out;
    animation-fill-mode: both;
}

.nav-item:nth-child(1) { animation-delay: 0.1s; }
.nav-item:nth-child(2) { animation-delay: 0.2s; }
.nav-item:nth-child(3) { animation-delay: 0.3s; }
.nav-item:nth-child(4) { animation-delay: 0.4s; }
.nav-item:nth-child(5) { animation-delay: 0.5s; }
.nav-item:nth-child(6) { animation-delay: 0.6s; }
.nav-item:nth-child(7) { animation-delay: 0.7s; }
.nav-item:nth-child(8) { animation-delay: 0.8s; }
.nav-item:nth-child(9) { animation-delay: 0.9s; }
.nav-item:nth-child(10) { animation-delay: 1.0s; }

@keyframes slideInLeft {
    from {
        opacity: 0;
        transform: translateX(-20px);
    }
    to {
        opacity: 1;
        transform: translateX(0);
    }
}
```

**Actualizar el media query de prefers-reduced-motion:**
```css
@media (prefers-reduced-motion: reduce) {
    .page-transition-out,
    .page-transition-in,
    .nav-link::before,
    .sidebar,
    .sidebar-overlay {
        animation: none !important;
        transition: none !important;
    }
}
```

### Animaciones que se mantienen:
- ✅ **Transiciones de página** (`page-transition-out`, `page-transition-in`)
- ✅ **Efecto hover mejorado** (brillo deslizante en `.nav-link::before`)
- ✅ **Indicador activo con animación** (barra lateral azul)
- ✅ **Animaciones de sidebar móvil** (transform y opacity)
- ✅ **Optimización de rendimiento** (`will-change`)
- ✅ **Accesibilidad** (`prefers-reduced-motion`)

### Resultado esperado:
- Los items del sidebar aparecerán instantáneamente sin animación cascada
- Las demás animaciones seguirán funcionando normalmente
- La experiencia de usuario será más limpia y profesional
